## Week 11: Days 71–77

### Typing Practice (10 min/day): Speed Drills

You have the technique. Now build speed through deliberate speed drilling.

**Method: Threshold training**
1. Find your comfortable accurate speed (say, 40 WPM)
2. Set the tutor 5–10 WPM above it (45–50 WPM)
3. Type at that speed — you'll make more errors
4. The errors force your fingers to find more efficient paths
5. When that speed becomes comfortable, raise it again

Use [monkeytype.com](https://monkeytype.com) — set to "words" mode, 1-minute tests, track weekly average.

**C code speed drills:** Create a file `drills/c-patterns.c` and type it from scratch each session. Try to beat your previous time:

```c
/* Common C idioms speed drill */

/* Null checks */
if (ptr == NULL) { return -1; }
if (!ptr) { return -EINVAL; }

/* String operations */
size_t len = strlen(s);
strncpy(dst, src, sizeof(dst) - 1);
dst[sizeof(dst) - 1] = '\0';

/* Error handling pattern */
int ret = do_something();
if (ret < 0) {
    fprintf(stderr, "error: %s\n", strerror(-ret));
    goto cleanup;
}

/* Array iteration */
for (size_t i = 0; i < ARRAY_SIZE(arr); i++) {
    process(&arr[i]);
}

/* Bit manipulation */
#define BIT(n)          (1UL << (n))
#define SET_BIT(x, n)   ((x) |= BIT(n))
#define CLR_BIT(x, n)   ((x) &= ~BIT(n))
#define TST_BIT(x, n)   ((x) & BIT(n))
```

### Emacs Practice (10 min/day): Keyboard Macros

Keyboard macros let you record and replay sequences of keystrokes. For C programming, they're excellent for repetitive structural edits.

| Command | Keys | What it does |
|---------|------|-------------|
| Start recording | `C-x (` | Begin macro recording |
| Stop recording | `C-x )` | End macro recording |
| Play macro | `C-x e` | Execute last macro |
| Play macro N times | `C-u 5 C-x e` | Execute macro 5 times |
| Name macro | `M-x name-last-kbd-macro` | Give macro a name |
| Insert macro as elisp | `M-x insert-kbd-macro` | Write it to init.el |

**Example: Adding a `\n` to end of each line in a block**

1. `C-x (` — start recording
2. `C-e` — go to end of line
3. Type ` \` — add backslash continuation (or whatever your pattern)
4. `C-n` `C-a` — go to start of next line
5. `C-x )` — stop recording
6. `C-u 20 C-x e` — replay 20 times

**Days 71–74: Occur and multi-file search**

| Command | Keys | What it does |
|---------|------|-------------|
| Occur | `M-x occur` | Show all lines matching regexp |
| Multi-occur | `M-x multi-occur-in-matching-buffers` | Occur across multiple files |
| Grep | `M-x grep` | Run grep, navigate results |
| Rgrep | `M-x rgrep` | Recursive grep with prompts |

**Occur is underrated.** Type `M-x occur`, enter `malloc`, and you get a buffer showing every line in the current file with `malloc`, with links. Click any result to jump there. For code review, this is invaluable.

**Days 75–77: Query-replace with regexp**

| Command | Keys | What it does |
|---------|------|-------------|
| Replace string | `M-x replace-string` | Simple find/replace |
| Query replace | `M-%` | Interactive find/replace |
| Query replace regexp | `C-M-%` | Regexp find/replace |

In query-replace:
- `y` — replace this one
- `n` — skip this one
- `!` — replace all remaining
- `q` — quit
- `C-r` — recursive edit at this spot (edit, then `C-M-c` to return)

**Regexp example for C:** Change all `malloc(sizeof(Type))` to `malloc(sizeof(*ptr))`:
```
C-M-%
Find: malloc(sizeof(\([A-Za-z_]+\)))
Replace: malloc(sizeof(*\1))    ← \1 is the captured group
```

### Emacs Practice (10 min/day): Org-Mode

Org-mode is a major mode for structured text, outlining, task management, and literate programming. Even if you use it only for notes and TODOs, it repays learning.

**Days 78–80: Org-mode basics**

Create `~/notes/dev.org`:
```
C-x C-f ~/notes/dev.org
```

Org-mode basics:
```org
* Project Notes                     ← First-level heading (*)
** Active Tasks                     ← Second-level (**)  
*** TODO Fix the memory leak        ← Third level, TODO keyword
*** DONE Refactor parse_header()    ← Done item
    CLOSED: [2024-01-15]

** Code Notes
*** malloc pattern
    Always use sizeof(*ptr) not sizeof(Type):
    #+BEGIN_SRC c
    Node *n = malloc(sizeof(*n));   /* Good */
    Node *n = malloc(sizeof(Node)); /* Fragile */
    #+END_SRC
```

| Key | What it does |
|-----|-------------|
| `TAB` | Cycle heading expand/collapse |
| `S-TAB` | Cycle all headings |
| `M-<RET>` | New heading at same level |
| `M-<right>` | Demote heading (deeper level) |
| `M-<left>` | Promote heading |
| `C-c C-t` | Cycle TODO state |
| `C-c C-s` | Schedule a task |
| `C-c C-d` | Set deadline |
| `C-c a` | Open agenda view |
| `C-c C-c` | Execute code block (in src blocks) |

**Days 81–84: Org for C development**

Use org-mode as a project notebook:

```org
* libfoo Development

** Build & Test
   #+BEGIN_SRC sh
   make clean && make && ./test_suite
   #+END_SRC

** Known Issues
*** TODO Investigate crash in parse_escaped_string()
    Crash occurs when input contains null bytes.
    Backtrace:
    #+BEGIN_EXAMPLE
    #0  parse_escaped_string (s=0x0) at parser.c:142
    #1  parse_value (...)
    #+END_EXAMPLE

** API Reference Notes
   The foo_init() function must be called before any other foo_* functions.
   Returns 0 on success, negative errno on failure.
```

You can execute `#+BEGIN_SRC sh` blocks directly with `C-c C-c`. This turns your notes file into an executable document.
