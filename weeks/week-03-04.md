# Weeks 3–4: Top Row & Navigation

## Typing Goal by End of Week 4
- Full alphabet available without looking (still slow — that's fine)
- 20+ WPM on common English words
- Beginning to feel finger independence

## Emacs Goal by End of Week 4
- Search forward and backward through a file
- Use the mark/region system for cut/copy/paste
- Work with multiple buffers
- Use `M-x` to run commands by name

---

## Week 3: Days 15–21

### Typing Practice (10 min/day)

**Top row, left hand:** q, w, e, r, t  
**Top row, right hand:** y, u, i, o, p

Introduce them in pairs. Don't rush to do all of them at once.

**Days 15–17:** Add `e`, `i` (index finger reach up)
```
fie die lie defile defied idled filed
jedi feed eel eel seeds fields
```

**Days 18–21:** Add `r`, `u`, then `t`, `y`
```
furl fuel rule rude lure urged
fur rule rude duel fluid dueled
true rut utter utter turtle
```

Use Keybr.com's "smart" mode — it will focus on your weakest letters automatically. Or use gtypist lessons `T.1` through `T.4`.

### Emacs Practice (10 min/day)

**Days 15–17: Search**

| Command | Keys | What it does |
|---------|------|-------------|
| Incremental search forward | `C-s` | Search as you type (forward) |
| Incremental search backward | `C-r` | Search backward |
| Next match | `C-s` again | While searching, find next |
| Exit search (keep position) | `<RET>` or `C-g` | Stop searching |

**Using search for navigation:** This is how Emacs power users navigate. Instead of scrolling to a function, you type `C-s functionname` and you're there. Learn this habit now.

**Days 18–21: Cut/Copy/Paste (Emacs calls it Kill/Yank)**

Emacs terminology:
- **Kill** = cut (to kill ring, Emacs's clipboard)
- **Yank** = paste
- **Mark** = the other end of a selection

| Command | Keys | What it does |
|---------|------|-------------|
| Set mark | `C-<SPC>` | Start a selection here |
| Kill region | `C-w` | Cut the selected region |
| Copy region | `M-w` | Copy the selected region |
| Yank | `C-y` | Paste |
| Kill line | `C-k` | Cut to end of line |
| Yank previous | `M-y` | Cycle through kill ring after `C-y` |

**Exercise:** In a C file, select a function with `C-<SPC>`, navigate to the end of the function with `C-s` or `C-n`, then kill and yank it somewhere else.

## Week 4: Days 22–28

### Typing Practice (10 min/day)

**Days 22–25:** Add `w`, `o`, `q`, `p`
```
word flow power word flow plow

```

**Days 26–28:** Full top row consolidation. Type real words using all top row + home row keys.
```
worker perform flower purpose report worker

```

**Target: 20 WPM on mixed home+top row text.**

### Emacs Practice (10 min/day)

**Days 22–24: Multiple buffers**

A *buffer* in Emacs is an open file (or other content). You'll have many open at once.

| Command | Keys | What it does |
|---------|------|-------------|
| List buffers | `C-x C-b` | Show all open buffers |
| Switch buffer | `C-x b` | Switch to buffer by name |
| Kill buffer | `C-x k` | Close a buffer |
| Next buffer | `C-x <right>` | Cycle to next buffer |
| Prev buffer | `C-x <left>` | Cycle to previous buffer |

**Days 25–28: Windows (splits)**

In Emacs, a *window* is a pane showing a buffer. Multiple windows = split screen.

| Command | Keys | What it does |
|---------|------|-------------|
| Split horizontally | `C-x 2` | Split into top and bottom |
| Split vertically | `C-x 3` | Split into left and right |
| Other window | `C-x o` | Move to the other window |
| Close other windows | `C-x 1` | Maximize current window |
| Close this window | `C-x 0` | Close current window/pane |

**Exercise:** Open two C files side by side with `C-x 3`. Use `C-x o` to switch between them. This is how you work with a header and implementation simultaneously.

**Days 26–28: Magit — Git from inside Emacs, starting now**

Magit is introduced here deliberately early. If you're logging your progress in this repo (which you should be), you want to commit from inside Emacs from day one — not context-switch to a terminal for git. It's also the thing that most often makes people say "okay, I'm staying."

Install it by uncommenting the Magit section in your `init.el` (see `resources/init-el-starter.md`), then restart Emacs.

**The core workflow — four keys:**

```
C-x g    → open Magit status buffer
s        → stage file or hunk under cursor
c c      → commit (C-c C-c to confirm the message)
q        → quit Magit
```

Add pushing: `P p` → push to remote. That's your complete daily git workflow without ever leaving Emacs.

Inside Magit, press `TAB` on any file to expand its diff into individual hunks — you can stage a single hunk with `s` rather than the whole file. Selective staging is painful in CLI git and trivial here.

**Exercise:** Make a change to your progress log. Open Magit with `C-x g`, expand the diff, stage it, commit with a message, push. All without leaving Emacs.

**Days 26–28 also: M-x — the command launcher**

`M-x` lets you run any Emacs command by name. Your escape hatch when you forget a keybinding.

```
M-x replace-string        → find and replace
M-x goto-line             → jump to line number
M-x sort-lines            → sort selected lines
M-x whitespace-cleanup    → clean up trailing whitespace
```

With `which-key` installed, pressing `C-x` and pausing shows available completions. Use this constantly.

## WPM Check

By end of week 4, test yourself on a standard typing test (use 1-minute test on [monkeytype.com](https://monkeytype.com) or similar). Record it in your log.

Expected range: **15–25 WPM** — don't be discouraged if you're at the low end. You're building correct habits, not raw speed. Speed comes later.
