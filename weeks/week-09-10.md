## Week 9: Days 57–63

### Typing Practice (10 min/day): C Operators & Patterns

This week is almost entirely C-specific symbol drilling. Use real code patterns, not abstract drills.

**Days 57–58: Comparison and logical operators**
```
== != < > <= >=
&& || !
if (x == 0 && y != NULL)
while (i < n && arr[i] != target)
if (!initialized || size <= 0)
```

**Days 59–60: Bitwise and shift operators**
```
& | ^ ~ << >>
flags & MASK
status |= FLAG_SET
bits ^= toggle
value << 3
byte >> 4
uint32_t mask = (1u << n) - 1;
```

**Days 61–63: Preprocessor and declaration patterns**
```
#include <stdio.h>
#include "myheader.h"
#define MAX(a, b) ((a) > (b) ? (a) : (b))
#ifndef HEADER_H
#define HEADER_H
#endif /* HEADER_H */

typedef struct {
    int x;
    int y;
} Point;

static inline int clamp(int v, int lo, int hi) {
    return v < lo ? lo : v > hi ? hi : v;
}
```

Type these patterns. Slowly. Without looking. Every time you miss a `{` or `;`, note which finger hesitated. Practice that specific key.

### Emacs Practice (10 min/day): C-Mode

**Days 57–59: C-mode fundamentals**

C-mode activates automatically when you open `.c` or `.h` files.

| Command | Keys | What it does |
|---------|------|-------------|
| Indent line | `<TAB>` | Indent current line correctly |
| Indent region | `C-M-\` | Indent selected region |
| Fill paragraph | `M-q` | Rewrap comment block |
| Forward function | `C-M-e` | Move to end of current function |
| Beginning of function | `C-M-a` | Move to start of current function |
| Mark function | `C-M-h` | Select entire current function |
| Forward sexp | `C-M-f` | Move forward over balanced expression |
| Backward sexp | `C-M-b` | Move backward over balanced expression |

**`C-M-f` and `C-M-b` are extremely useful for C.** They jump over matching parens, brackets, and braces. Navigate `if (complex && condition)` by moving over the whole parenthesized expression.

**Days 60–61: Indentation style**

Emacs C-mode supports multiple indent styles. Set yours in init.el:

```elisp
;; Choose one:
(setq c-default-style "linux")    ; Linux kernel style (tabs, 8-wide)
(setq c-default-style "k&r")      ; K&R style  
(setq c-default-style "gnu")      ; GNU style (default)
(setq c-default-style "bsd")      ; BSD/Allman style
```

**Days 62–63: ctags / xref — navigating large codebases**

First, generate tags for your project:

```bash
# In your project root (in eshell or term):
ctags -R .          # generates TAGS file
# or for etags (Emacs native format):
find . -name "*.c" -o -name "*.h" | xargs etags
```

Then in Emacs:

| Command | Keys | What it does |
|---------|------|-------------|
| Find tag (jump to definition) | `M-.` | Jump to definition of symbol at point |
| Return from tag | `M-,` | Jump back after following a tag |
| Find tag in other window | `C-x 4 .` | Open definition in split window |
| List tags matching | `M-x tags-apropos` | Find all tags matching pattern |

**Exercise:** In a multi-file C project, put cursor on a function call and press `M-.`. Jump to its definition. Press `M-,` to return. Repeat until it's automatic. This replaces `grep` + manual navigation.

## Week 10: Days 64–70

### Typing Practice (10 min/day): Speed on Real C Code

By now you have all the keys. This week is speed-building on realistic C code.

**Drill:** Type the following programs (or fragments of your own real code) touch-style. Time yourself.

**Target: Get through this in under 3 minutes (increasing speed):**
```c
#include <stdlib.h>
#include <string.h>

typedef struct node {
    int value;
    struct node *next;
} Node;

Node *node_create(int value) {
    Node *n = malloc(sizeof(*n));
    if (n == NULL) return NULL;
    n->value = value;
    n->next = NULL;
    return n;
}

void list_push(Node **head, int value) {
    Node *n = node_create(value);
    if (n == NULL) return;
    n->next = *head;
    *head = n;
}

void list_free(Node *head) {
    while (head != NULL) {
        Node *next = head->next;
        free(head);
        head = next;
    }
}
```

Note your time each day. You'll improve noticeably across the week.

### Emacs Practice (10 min/day): GDB & Magit

**Days 64–66: GDB inside Emacs**

Emacs has a built-in GDB interface that shows source, stack frames, breakpoints, and locals in synchronized windows.

```
M-x gdb                     → start GDB (prompts for program)
M-x gdb-many-windows        → switch to full GDB layout
```

Basic GDB commands (same as regular GDB, entered in the GDB buffer):
```
run                         → start the program
break myfile.c:42           → set breakpoint at line 42
break function_name         → set breakpoint at function
next  (or n)                → step over
step  (or s)                → step into
continue  (or c)            → continue
print variable              → print variable value
backtrace  (or bt)          → show call stack
quit                        → exit GDB
```

**In Emacs GDB mode:** You can click in the source buffer to set breakpoints, and local variables appear in a separate pane.

**Days 67–70: Magit — Git inside Emacs**

Magit is widely considered the best Git interface ever built. Install it:

```elisp
;; In your init.el (if using package.el):
(unless (package-installed-p 'magit)
  (package-install 'magit))

(global-set-key (kbd "C-x g") 'magit-status)
```

**The Magit status buffer:**

```
C-x g            → open Magit status
```

Inside Magit status:
| Key | What it does |
|-----|-------------|
| `s` | Stage file or hunk |
| `u` | Unstage |
| `c c` | Commit (opens commit message buffer) |
| `P p` | Push |
| `F p` | Pull |
| `b b` | Switch branch |
| `b c` | Create new branch |
| `l l` | Show log |
| `d d` | Diff |
| `TAB` | Expand/collapse section |
| `q` | Quit Magit |

**The killer feature:** In the status buffer, with a file's changes expanded, you can press `s` on individual *hunks* (diff sections) to stage only parts of a file. This is difficult in CLI git and trivial in Magit.

**Exercise:** Make changes to a C file, open Magit with `C-x g`, stage individual hunks, write a commit message, and push — all without leaving Emacs.
