### Typing Practice (10 min/day)

**The number row:** 1234567890

Number row requires significant upward reach, especially on the left. The right hand has an easier reach to 7–0.

**Finger assignments (standard):**
```
1 → left pinky
2 → left ring
3 → left middle
4 → left index
5 → left index (stretch)
6 → right index (stretch)
7 → right index
8 → right middle
9 → right ring
0 → right pinky
```

### Emacs Practice (10 min/day): Shell Integration

This is where Emacs starts to feel like home for command-line people.

**Days 43–45: Shell buffers**

| Command | Keys / Command | What it does |
|---------|---------------|-------------|
| Shell buffer | `M-x shell` | Run your login shell in a buffer |
| Eshell | `M-x eshell` | Emacs's built-in Lisp shell |
| Term | `M-x term` | Full terminal emulator |
| Ansi-term | `M-x ansi-term` | Better full terminal |

**Eshell is special** — it's an Emacs Lisp shell that understands both shell syntax and Emacs commands. You can pipe output to Emacs buffers, use Lisp as your scripting language, and run Emacs commands inline.

```bash
# In eshell, these all work:
ls -la
find . -name "*.c" | sort
grep -r "malloc" src/ > #<buffer grep-results>   # output to buffer!
cat myfile.c | wc -l
```

**Days 46–49: Eshell deeper**

In eshell:
```bash
# Open a file in Emacs from eshell
find-file myfile.c

# Switch to a buffer
switch-to-buffer *scratch*

# Run elisp directly
(+ 2 3)          → 5

# History navigation
M-p / M-n        → previous/next command (same as in regular Emacs)
C-r              → reverse history search

# Glob patterns
ls *.c
rm **/*.o       # recursive glob!
```

**Exercise:** Open eshell, navigate to a C project, run `make` or `gcc` directly from eshell. Watch the output appear in the buffer. Use `C-x o` to switch to your C file and back.

## Week 8: Days 50–56

### Typing Practice (10 min/day): C Symbol Drills

Now we attack the symbols that C programmers use constantly. These are the keys that typing tutors ignore and hunt-and-peckers find most painful.

**Symbol finger assignments:**
```
!  → left pinky shift + 1
@  → left ring shift + 2
#  → left middle shift + 3
*  → right middle shift + 8
(  → right ring shift + 9
)  → right pinky shift + 0
-  → right pinky (no shift)
_  → right pinky shift + -
=  → right pinky (stretch)
+  → right pinky shift + =
[  → right pinky (stretch right)
]  → right pinky (far stretch)
{  → right pinky shift + [
}  → right pinky shift + ]
;  → right pinky (home row!)
:  → right pinky shift + ;
'  → right pinky (next to ;)
/  → right pinky (bottom row)
\  → right pinky (far right)
&  → right index shift + 7
|  → right pinky shift + \
<  → right ring shift + comma
>  → right middle shift + period   ← wait, no:
                                    comma = right middle, period = right ring
                                    < = right middle shift + comma
                                    > = right ring shift + period
```

**Days 50–52:** Parentheses, brackets, braces — the grouping family
```
() [] {} () [] {} () [] {}
(int x) [MAX] {0}
(struct node *) malloc(sizeof(struct node))
if (x > 0) { return x; }
for (int i = 0; i < n; i++) { arr[i] = 0; }
```

**Days 53–56:** Pointer and arithmetic operators
```
*ptr &var ptr->field (*ptr).field
int *p = NULL;
char *s = "hello";
struct foo *bar = malloc(sizeof(*bar));
bar->value = 42;
free(bar);
```

### Emacs Practice (10 min/day): Compilation & Error Navigation

**Days 50–52: Compile mode**

| Command | Keys | What it does |
|---------|------|-------------|
| Compile | `M-x compile` | Run a compile command |
| Recompile | `M-x recompile` or `C-x `\`` | Rerun last compile command |
| Next error | `M-g n` or `C-x \`` | Jump to next compile error |
| Previous error | `M-g p` | Jump to previous error |

**Setup:** The first time you run `M-x compile`, it will prompt for a command. Type `make` or `gcc -Wall -g myfile.c -o myfile`. Emacs remembers this for `M-x recompile`.

The compile buffer shows errors as links. You can click them (or press Enter on them) to jump directly to the file and line with the error. This is the fundamental C development workflow in Emacs.

**Days 53–56: Compile workflow**

Practice this cycle:
1. Edit C file
2. `M-x recompile` (or bind it: see init.el starter)  
3. If errors: `C-x \`` to jump to first error
4. Fix it
5. `C-x \`` again for next error
6. When clean: run your program in eshell

**Suggested keybinding to add to init.el:**
```elisp
(global-set-key (kbd "<f5>") 'recompile)
(global-set-key (kbd "<f6>") 'next-error)
```

## Laptop Keyboard Variations Note

By now you've probably noticed differences across your laptops:
- Some have `Fn` rows that behave differently
- Some have different depths/travel
- Some `Ctrl` key placement varies even after Caps Lock remap

See `cheatsheets/keyboard-variations.md` for specific strategies. The key insight: **touch typing is about finger positions relative to home row, not absolute key positions.** Once your home row anchoring is solid (F and J bumps), everything else follows relatively. The variation becomes less important.
