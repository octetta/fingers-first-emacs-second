# Weeks 1–2: Home Row & Emacs Survival

## Typing Goal by End of Week 2
- Home row keys (asdf jkl;) without looking: confident, no hesitation
- 15+ WPM on home-row-only drills
- Sitting position and hand placement: correct and consistent

## Emacs Goal by End of Week 2
- Open Emacs without panicking
- Open a file, edit it, save it, quit
- Move the cursor without arrow keys
- Undo a mistake

---

## The Setup (Do This First)

### Remap Caps Lock to Ctrl — on every machine
This is the single most important configuration for Emacs comfort.

**Linux (X11):** Add to `~/.Xmodmap`:
```
remove Lock = Caps_Lock
add Control = Caps_Lock
```
Or use `setxkbmap -option ctrl:nocaps` in your shell config.

**Linux (Wayland/GNOME):** Settings → Keyboard → Special Character Entry → Ctrl key position → Caps Lock as Ctrl

**macOS:** System Preferences → Keyboard → Modifier Keys → Caps Lock → Ctrl

**Windows:** Use AutoHotkey or SharpKeys.

Do this *before* your first Emacs session. Without it, your left pinky will be reaching to the bottom-left corner constantly.

### Install Emacs
See `resources/tools.md` for platform-specific recommendations. Use Emacs 29+ if possible.

### Install the starter config
Copy `resources/init-el-starter.md` to `~/.emacs.d/init.el`. This gives you:
- Line numbers
- A sane color theme
- `which-key` (shows what keybindings are available)
- Basic C-mode configuration

## Hand Position: The Foundation

```
Left hand:              Right hand:
  A  S  D  F              J  K  L  ;
  ↑  ↑  ↑  ↑              ↑  ↑  ↑  ↑
 pin mid idx idx        idx mid rin pin
```

- **Wrists off the desk** (or barely touching)
- **Fingers curved**, resting lightly on home row
- **F and J keys have bumps** — use them to anchor without looking
- **Thumbs** rest on spacebar
- **Look at the screen, not your hands**

Cover your hands with a cloth if you catch yourself looking. Seriously. Do it.

## Week 1: Days 1–7

### Typing Practice (10 min/day)

**Tool:** Use [Keybr.com](https://www.keybr.com) or [TypeFu](https://type-fu.com) set to home-row-only mode, or just use `gtypist` from the command line:

```bash
# Install gtypist
sudo apt install gtypist    # Debian/Ubuntu
brew install gtypist        # macOS

# Run lesson D.1 (home row)
gtypist
```

**Daily drill — type these without looking, slowly:**
```
asdf jkl; asdf jkl;
aaa sss ddd fff jjj kkk lll ;;;
fads jads fads jads
flask flak; glad dials
```

**Week 1 typing focus:** Only f, d, s, a, j, k, l, ;  
Speed doesn't matter. Accuracy does. Go as slow as you need to not make mistakes.

### Emacs Practice (10 min/day)

**Day 1–2: Launch it and navigate to your work**

Rather than opening Emacs directly on a file, start by launching Emacs and using Dired — the built-in directory browser — to navigate to your file. This is closer to how you'll actually work.

```bash
emacs .        # open Emacs in current directory — lands in Dired
```

You'll see a file listing. Basic Dired survival:

| Key | What it does |
|-----|-------------|
| `n` / `p` | Move down / up in the listing |
| `<RET>` | Open file or directory |
| `^` | Go to parent directory |
| `q` | Quit Dired (back to previous buffer) |

Navigate to a C file and press `<RET>` to open it. You're now editing. This is the natural Emacs entry point — filesystem first, file second.

Essential survival commands — learn these alongside Dired:

| Command | Keys | What it does |
|---------|------|-------------|
| Open file | `C-x C-f` | Find (open) a file |
| Save file | `C-x C-s` | Save current buffer |
| Quit Emacs | `C-x C-c` | Exit (prompts to save) |
| Cancel | `C-g` | Cancel any command — your escape hatch |
| Undo | `C-/` or `C-x u` | Undo last change |

`C-g` is sacred. When Emacs does something confusing, `C-g` gets you out.

**Day 3–5: Moving without arrow keys**

| Command | Keys | What it does |
|---------|------|-------------|
| Forward char | `C-f` | Move right one character |
| Backward char | `C-b` | Move left one character |
| Next line | `C-n` | Move down one line |
| Prev line | `C-p` | Move up one line |
| Beginning of line | `C-a` | Jump to start of line |
| End of line | `C-e` | Jump to end of line |

Notice: all of these are on or near the home row. This is intentional. Emacs was designed so your hands never need to leave.

**Exercise:** Open a C file (or create `practice.c`). Navigate around it using ONLY `C-f`, `C-b`, `C-n`, `C-p`. No arrow keys. No mouse. It will feel slow. That's fine.

**Day 6–7: Basic editing**

| Command | Keys | What it does |
|---------|------|-------------|
| Delete char forward | `C-d` | Delete character under cursor |
| Delete char backward | `<DEL>` (backspace) | Delete character before cursor |
| Kill line | `C-k` | Delete from cursor to end of line |
| New line | `C-j` or `<RET>` | Insert newline |

## Week 2: Days 8–14

### Typing Practice (10 min/day)

Add the `g`, `h` keys this week — they're the reach keys off the home row center.

```
glad hall gash dash flag jag lag
had half glad shag flag dash
```

Continue on Keybr.com or gtypist. Set it to introduce `g` and `h` only.

**Week 2 typing focus:** Solidify home row. Introduce g and h. Target: 15 WPM accurately.

### Emacs Practice (10 min/day)

**Day 8–10: Word-level movement**

| Command | Keys | What it does |
|---------|------|-------------|
| Forward word | `M-f` | Move forward one word |
| Backward word | `M-b` | Move backward one word |
| Kill word | `M-d` | Delete word forward |
| Kill word back | `M-<DEL>` | Delete word backward |

`M-` means Meta, which is the Alt key (or Esc if Alt doesn't work on your laptop).

**Day 11–12: Help system**

Emacs has incredible built-in documentation. Learn to use it now.

| Command | Keys | What it does |
|---------|------|-------------|
| Describe key | `C-h k` then a key | What does this key do? |
| Describe function | `C-h f` | Describe a function by name |
| Apropos | `C-h a` | Search for commands matching a word |
| Tutorial | `C-h t` | The built-in Emacs tutorial |

**Do the built-in tutorial at least once.** Run `C-h t`. It's old-fashioned but comprehensive. With your new touch-typing practice, work through it without looking at your keyboard.

**Day 13–14: Your first real session**

Use Emacs for 30 minutes to edit a real C file you're working on. No other editor.

Start by launching `emacs .` and navigating via Dired. Use `C-x d` to open Dired at any point:

```
C-x d    → open Dired (prompts for directory)
```

A few more Dired keys worth adding this week:

| Key | What it does |
|-----|-------------|
| `C-x C-f` | Open file by path (when you know exactly where it is) |
| `C-x d` | Open Dired at a directory |
| `g` | Refresh the Dired listing |
| `m` / `u` | Mark / unmark a file |

You'll get the full Dired treatment in weeks 5–6. For now: use it to navigate, open files, and get comfortable with the idea that the filesystem is just another buffer.

## Troubleshooting Week 1–2

**"I keep looking at my keyboard"**  
Put a piece of paper or thin cloth over your hands. Physical barrier beats willpower.

**"The Emacs cursor keys feel impossibly slow"**  
They are. You'll add word-level and page-level navigation next week. For now, embrace the slowness — it's building the correct habit.

**"My left pinky hurts from Ctrl"**  
You haven't remapped Caps Lock yet. Do it now. See setup section above.

**"Emacs opened in terminal mode, not GUI"**  
That's fine. Terminal Emacs (`emacs -nw`) is excellent and more portable. The keybindings are identical.

**"I accidentally closed Emacs and lost my work"**  
Emacs auto-saves to `#filename#`. You can recover with `M-x recover-file`. Also: save often with `C-x C-s`. Make it automatic muscle memory this week.
