## Weekly Emacs Milestone Tracker

Check off when you've genuinely internalized each skill (not just "tried it once"):

### Phase 1: Survive (Weeks 1–2)
- [ ] Caps Lock remapped to Ctrl on all machines
- [ ] Can open a file: `C-x C-f`
- [ ] Can save a file: `C-x C-s`
- [ ] Can quit: `C-x C-c`
- [ ] Reach for `C-g` when lost
- [ ] Navigate with `C-f`, `C-b`, `C-n`, `C-p` (no arrow keys)
- [ ] Undo with `C-/`
- [ ] Completed Emacs built-in tutorial (`C-h t`)

### Phase 2: Expand (Weeks 3–4)
- [ ] Word navigation: `M-f`, `M-b`
- [ ] Search: `C-s` and `C-r`
- [ ] Cut/copy/paste: `C-w`, `M-w`, `C-y`
- [ ] Set mark: `C-<SPC>`
- [ ] Multiple buffers: `C-x b`, `C-x C-b`, `C-x k`
- [ ] Split windows: `C-x 2`, `C-x 3`, `C-x o`
- [ ] Run commands by name: `M-x`

### Phase 3: Build (Weeks 5–6)
- [ ] Page navigation: `C-v`, `M-v`, `M-<`, `M->`
- [ ] Navigate Dired: `C-x d`
- [ ] File operations in Dired: rename, copy, delete
- [ ] Use registers for bookmarks
- [ ] Jump to line: `M-g g`

### Phase 4: Code (Weeks 7–8)
- [ ] Number row without looking
- [ ] Common C symbols: `{}`, `[]`, `()`, `*`, `&`, `->` without looking
- [ ] Eshell: `M-x eshell`
- [ ] Output to Emacs buffer from eshell
- [ ] Run compile: `M-x compile`
- [ ] Navigate errors: `C-x \``
- [ ] Recompile shortcut working: `<f5>`

### Phase 5: Specialize (Weeks 9–10)
- [ ] C expression navigation: `C-M-f`, `C-M-b`, `C-M-a`, `C-M-e`
- [ ] Jump to definition: `M-.` (tags or eglot)
- [ ] Return from tag: `M-,`
- [ ] Header/source switching: `C-c o`
- [ ] GDB session inside Emacs
- [ ] Magit: stage, commit, push

### Phase 6: Accelerate (Weeks 11–12)
- [ ] Keyboard macro: record and play back
- [ ] Occur: `M-x occur`
- [ ] Query-replace regexp: `C-M-%`
- [ ] Org-mode: basic headings and TODO items
- [ ] Org-mode: capture a note with `C-c c`

### Phase 7: Integrate (Week 13)
- [ ] Personal `init.el` reflects my actual workflow
- [ ] Emacs is my default editor — I no longer reach for another tool
- [ ] Touch typing feels natural (not effortful)
- [ ] Typing speed still improving without structured drills
